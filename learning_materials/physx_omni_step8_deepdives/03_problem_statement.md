# 03 论文要解决的问题精讲

对应 `paper-reading.md`：`## 3. 论文要解决的问题`

## 论文原意

传统 3D 生成更关注视觉指标：

- 几何像不像。
- 纹理是否好看。
- 多视角是否一致。
- 渲染质量是否高。

但机器人和仿真任务还需要物理层面的信息：

- 物体真实尺寸。
- 材料物性。
- 哪些部位可以抓、按、推、拉。
- 是否有门、抽屉、轮子、滑轨、铰链。
- 关节轴和运动范围是否合理。
- 进入物理引擎后是否稳定、可碰撞、可交互。

PhysX-Omni 要解决的就是这个断层：从“视觉 3D”推进到“物理可用 3D”。

## 为什么旧方法不够

论文认为已有方法主要有两类缺口：

1. 很多 image-to-3D 方法只生成外观资产，不生成物理属性。
2. 已有 physical 3D 生成通常只覆盖某一类资产，例如只做 rigid、只做 articulated、只做 deformable，缺少统一框架。

这会导致生成模型看起来很强，但一放进仿真器就暴露问题：尺寸不对、材料不合理、部件不能动、关节错位、碰撞体缺失或语义无法用于机器人策略学习。

## 代码和数据落点

PhysX-Omni 的输出结构直接体现这个问题定义。官方 demo 生成的 `basic_info.json` 包含：

- `dimension`：尺寸。
- `parts`：部件。
- `material`、`density`、`Poisson's Ratio` 等物理属性。
- `group_info`：部件关系和运动结构。

这和普通 3D 生成只输出 `.obj` 或 `.glb` 不同。后者通常没有可驱动关节，也没有部件级物理语义。

## 大白话说明

如果你只想做展示图，一个漂亮 mesh 就够了。  
如果你要训练机器人抓取、推拉、开门、搬运，mesh 不够。

机器人需要知道：

- 这个把手能抓吗？
- 这个盖子能掀吗？
- 这个轮子能滚吗？
- 这个箱子多大多重？
- 这个材料摔到地上应该像金属还是塑料？

PhysX-Omni 就是试图让生成资产带上这些信息。

## 边界

单张图像无法精确反推出真实物理属性。论文解决的是“合理生成”和“可评估的 plausibility”，不是实验室级别的材料测量。复现时不要把材料参数理解成真实测量值，它们更接近常识和数据先验下的合理估计。
