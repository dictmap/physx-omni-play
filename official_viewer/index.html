Paper and code flow

论文主流程：从单图到物理资产

下面按真实运行链路组织，不按论文段落顺序组织。看代码时也应沿这条链路走。

01

图像理解与部件拆分

Qwen2.5-VL 读取输入图像，输出对象名、部件、材料/尺度描述、group_info 以及每个部件的 template RLE。

  • 1vlm_demo.py
  • basic_info.txt
  • coord_*.txt
02

RLE 到稀疏体素

把 template-based RLE 解码成 64³ 稀疏坐标。这里决定了“部件有没有体积”和“体积是否被压扁”。

  • decode_voxel_2drle_by_z()
  • ind_*.npy
  • allind.npy
03

TRELLIS 几何解码

将每个部件的稀疏坐标和条件图像送入 3D decoder，生成 textured GLB/OBJ/贴图。

  • 2infer_geo.py
  • decoder_each.py
  • to_glb()
04

物理结构装配

把部件、group_info、材料与关节假设写入 URDF 和 MuJoCo XML，形成可被仿真器消费的资产入口。

  • 3jsongen_update.py
  • basic.urdf
  • basic.xml
Core ideas

三个核心创新点怎么落地

这里对应 Step 3 和 supporting notes，重点看“论文说法”和“代码实际承载点”是否一致。

Template-based RLE 让 VLM 输出可解码体素

论文把复杂 3D 结构压成可由 VLM 生成的文本序列。好处是输出接口简单；风险是分辨率、拓扑复杂度和长序列稳定性。

打开精讲

PhysXVerse / PhysX-Bench 把评测从“像不像”推到“能不能用”

Bench 覆盖材料、尺度、affordance、运动学、描述一致性与渲染质量。关键争议在于 VLM judge 的稳定性和可复现性。

打开精讲

生成链路端到端，但每一段误差会传递

VLM 部件识别、RLE 体素、TRELLIS decoder、URDF/XML 装配彼此串联。M&M's 高罐偏矮就是体素阶段误差传到几何阶段的例子。
